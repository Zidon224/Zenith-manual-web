The easiest way to make a Zenith resource pack is in Textured module usingj son scripts


The best IDE text editor is Visual Studio Code

Get VS Code

Attributions for each element

Showing description/info about the resource pack

              "description": "Text to be desplayed in the description"

Showing image preview. Any format of image is accepted(bmp, png, jpg)

              "previewImage": "img.png"

              "whiteKey": "WhiteKey.png"

Key white (Pressed)

              "whiteKeyPressed": "WhiteKeyPressed.png"

              "middleTexture": "note.png"

Switches


Another feature that Zenith textured module has is to associate functions with switches of the type Drop down list.

This is much convenient and easy to control keys textures and other cutsomizations.


To include swicth tab in your resource pack this is the property for switches: (Down below)

            "switches": []

This switches must be lenght with a custom attribution to work.

Same width notes example switch:

            {
                "name": "Same width notes",
                "values": [
                  "Yes",
                  "No"
                ]
            }

Functions:

                "sameWidthNotes": {
                "_switch": "Same width notes",
                "Yes": "true",
                "No": "false"
            },

Textures switch chooser

Keyboard texture chooser

              {
                  "name": "Keyboard version",
                  "values": [
                    "V1 ",
                    "V2"
              }

For changing the entire keyboard version this are all of the properties that you have to write in your code.(Black keys pressed unpressed , White keys pressed unpressed , bar is optional, add it only if you have a keyboard with a divider top bar.)

It is also possible to create file paths to that image


                //Black key section
                    "blackKey": {
                    "_switch": "Keyboard version",
                    "V1": "path/to/blackKeyV1.png",
                    "V2": "path/to/blackKeyV2.png"
                },

                //Key white section
                    "whiteKey": {
                    "_switch": "Keyboard version",
                    "V1": "path/to/whiteKeyV1.png",
                    "V2": "path/to/whiteKeyV2.png"
                },

                //Black key pressed section
                    "blackKeyPressed": {
                    "_switch": "Keyboard version",
                    "V1": "path/to/blackKeyPressedV1.png",
                    "V2": "path/to/blackKeyPressedV2.png"
                },

                //White key pressed section
                    "whiteKeyPressed": {
                    "_switch": "Keyboard version",
                    "V1": "path/to/whiteKeyPressedV1.png",
                    "V2": "path/to/whiteKeyPressedV2.png"
                },

                //Note section (Middle texture)
                    "middleTexture": {
                    "_switch": "Keyboard version",
                    "V1": "path/to/noteV1.png",
                    "V2": "path/to/noteV2.png"
                },
                  
                //Bottom note edges
                    "bottomTexture": {
                    "_switch": "Keyboard version",
                    "V1": "path/to/noteEdgeV1.png",
                    "V2": "path/to/noteEdgeV2.png"
                },

                //Top note edges
                    "topTexture": {
                    "_switch": "Keyboard version",
                    "V1": "path/to/noteEdgeV1.png",
                    "V2": "path/to/noteEdgeV2.png"
                }

Is also possible to write some text under or above the switches for some info, sugestions or other things.

             {
                 "text": "Some text thing"
             }
